**Manage** and control your AI security by composing sovereign AI systems that contain internal and external models, MCP servers, and connections to applications.

**Govern**, monitor and audit your AI systems by applying AI governance policies that aligns to NIST, OWASP and OMB out of the box.

**Deploy** AI agents that automatically inherit your system-wide governance and integrate directly with your sovereign AI systems.

## Deployment Options

Prediction Guard's Unified CX supports flexible AI System deployment across different environments:

- **On-Premises**: Deploy in your own data center with Kubernetes or single-node binary
- **Cloud**: Deploy on AWS, Azure, or Google Cloud with managed Kubernetes
- **Air-Gapped**: Deploy in isolated environments with offline packages

## Platform Capabilities

Your deployed Prediction Guard platform provides:

- **Model Management**: Deploy curated, scanned and tested models from Hugging Face
- **System Management**: Create and manage multiple systems across environments
- **Security & Compliance**: Built-in security scanning, audit logs, and compliance features
- **API Management**: Create and manage API keys with granular permissions
- **Integrations**: Connect your AI systems to applications, knowledge bases and MCP servers that will be inherited by the agents that you deploy
- **Governance**: Apply AI governance policies across your systems that aligns with NIST, OWASP and OMB standards
- **Monitoring**: Real-time monitoring and alerting for your AI infrastructure

## Getting Started

<Steps toc={true}>

### Create your first sovereign AI system in the Unified CX

Start by creating your system in the Unified CX:

1. **Navigate to your Unified CX instance (e.g. [admin.predictionguard.com](https://admin.predictionguard.com))** and log in

2. **Go to Systems → Manage** and click **"Create System"** in the top-right corner

![Systems Page](./ux-screenshots/01-systems.png)

3. **Select "Quick Start"** for a simplified setup with sensible defaults

![Select Quick Start](./ux-screenshots/02-CreateSystem-QuickStart.png)

4. **Fill in your system details:**
   - **System Name**: A unique name for your system (e.g. `production`, `staging`)
   - **Public API Endpoint**: Enable this if you need access from outside your Kubernetes deployment, and add your TLS hostname(s)

![System Name and Public API Endpoint](./ux-screenshots/03-quicksystem-pg1.png)

5. **Click "Create System"** to create your AI system with default settings

### Choose your deployment method

Prediction Guard can be deployed anywhere that fits your needs. Choose the deployment method that works best for your environment:

#### On-Premises
Deploy in your own data center:
- [Kubernetes Cluster](/unified-cx/installation/on-premises/kubernetes) - Full Kubernetes deployment
- [Zero Dependency Binary](/unified-cx/installation/on-premises/binary) - Single node binary installation

#### Cloud Deployment
Deploy on major cloud providers:
- [AWS Deployment](/unified-cx/installation/cloud/aws) - Amazon Web Services
- [Azure Deployment](/unified-cx/installation/cloud/azure) - Microsoft Azure
- [GCP Deployment](/unified-cx/installation/cloud/gcp) - Google Cloud Platform

#### Air Gapped
Deploy in isolated environments. Note that air-gapped deployment requires **Custom** configuration instead of Quick Start during system creation:
- [Air Gapped Deployment](/unified-cx/installation/air-gapped/air-gapped) - Offline deployment guide

### Deploy your AI system

Follow the specific deployment guide for your chosen environment. When ready, click the **Deploy** button on your system in the Unified CX to get the installation command.

![Deploy Command](./ux-screenshots/04-deploy-kubectl.png)

The deployment process will:

1. **Generate an installation command** scoped to your system from the Unified CX
2. **Run the command** on a machine with `kubectl` access to your cluster
3. **Bootstrap your system** — Prediction Guard services will start up in the `predictionguard` namespace
4. **Verify the deployment** — your system will show as **Healthy** in the Unified CX once complete

### Access your Unified CX dashboard

Once deployed, your system is fully manageable from the Unified CX dashboard.

![Unified CX Dashboard](./ux-screenshots/06-system-deployed-manage-screen.png)

From here you can manage API keys, deploy models, configure MCP servers, apply governance policies, and monitor your system.

### Deploy your first model

From your system's management dashboard, navigate to **Models** and click **Add Private Model**.

![Models Page](./ux-screenshots/08-models-private-models-1.png)

Browse the model catalog and click **Select** on the model you want to deploy.

![Select a Model](./ux-screenshots/08-models-private-models-select-models-2.png)

Configure the model's **General Settings** — name, replicas, and container image details.

![Configure Model - General Settings](./ux-screenshots/08-models-private-models-configure-gen-settings.png)

The **Model Parameters** have been tested and preconfigured according to the model card — CPU, memory, accelerator cards, token limits, and capabilities. Make sure your node is sized appropriately for the model you've selected.

![Configure Model - Parameters](./ux-screenshots/08-models-private-models-configure-model-params-makesureyournodeissizedtothis.png)

Click **Create Private Model** to deploy. Once running, test it via API calls using one of your API keys.

<Callout intent="info">
Prediction Guard also supports **Managed Models** (hosted by Prediction Guard with additional Managed Model license but no setup required) and **External Models** (connect to OpenAI, Anthropic, Google, AWS, Azure, and more). All three types inherit your system's governance policies and safeguards automatically. See the full [Model Management](/unified-cx/administration/model-management) guide.
</Callout>

### Create API keys and start building

Set up access for your applications:

1. **Create API keys** with appropriate permissions
2. **Configure rate limits** and usage quotas
3. **Test API access** with your deployed models
4. **Start building** secure, compliant AI applications

</Steps>
